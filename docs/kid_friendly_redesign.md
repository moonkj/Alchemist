# 컬러 믹스: 연금술사 — 아이 친화적 UI/UX 재설계 (Kid-Friendly Redesign)

> 작성자: UX/UI Senior Designer Teammate
> 작성일: 2026-04-22
> 사유: 유저 강한 피드백 — "밑에 검은색 3가지 상자는 뭐야? 각 메뉴·버튼이 너무 작아. 너무 딱딱하고 아이들 친화적으로 만들어야지"
> 범위: MinimalGameScene(OnGUI) 전체 화면 재설계 가이드
> 대상: 7~12세 주력(코어) + 13세+ 보조(서브). 손가락 두께/시지각 발달 수준 고려.
> 주의: 본 문서는 **가이드**이며 이번 라운드 코드 수정은 포함하지 않는다. 리더가 다음 라운드 TOP5를 선정하여 Coder Teammate 에 넘긴다.

## A. 진단 — 5대 문제점

### A-1. 팔레트 슬롯 "검은 상자 3개" 인식 문제 (최우선)
- **현 구현**: `ResetPaletteColors()` 에서 빈 슬롯 컬러를 `new Color(0.22f, 0.24f, 0.30f, 0.55f)` (진한 회-검 반투명) 로 표현. 라벨 없음, 아이콘 없음, 테두리 없음.
- **원인**: 어포던스(affordance) 부재. 유저가 그것이 "무엇인지" 학습할 시각 단서(헤더, 라벨, 힌트 텍스트, 반짝임)가 0개.
- **증상**: 7세 테스터 → "검은 상자가 뭐야? 눌러도 돼?" / 40대 피드백 → "UI 고장난 줄 알았다".
- **위치**: MinimalGameScene.cs L247~256 (ResetPaletteColors), L521~523 (채워진 슬롯), L1431 이하 OnGUI HUD.

### A-2. 버튼 크기 — 어린이 터치 친화 기준 미달
- 로비 ⚙/? 버튼: **44h × 52w** (L1445, L1451).
- 갤러리 CTA: **44h × 128w** (L1438).
- 설정 "뒤로": **40h × 84w** (L1591).
- Apple HIG 최소 44pt, Material 48dp 를 **간신히** 넘는 수준 — 어린이 손가락(평균 8~11mm, 성인 대비 1.2~1.5배 fat finger)에는 부족.
- 결과: 오탭(misfire) 증가, 좌절감, 학습 장벽.

### A-3. 딱딱한 기본 GUI.skin 폰트 + 직사각형 버튼
- `new GUIStyle(GUI.skin.button)` 기본 스킨 그대로 사용 (L1355~1357). Unity 기본 회색 박스 = "개발자 프로토타입" 느낌.
- **radius 0** 직사각, 그림자 없음, hover 피드백 없음, bold 폰트만 가끔 적용.
- 아이들에게는 "학교 시험지" 같은 경직된 분위기로 인식됨.

### A-4. 색감·톤 어둠톤 편중
- 배경: `#1A1E29` 영역 (L=0.08~0.22). 검정에 가까움.
- 패널: `#0F1218` @ 0.92 (neutral.900). overlay 딤까지 합쳐지면 화면 60%가 먹색.
- 블록은 채도 높은데 배경이 어둡다보니 "어둠 속 네온사인" 분위기 → SF/호러 톤. **아이 친화와 정반대**.

### A-5. 이모지 활용 미미, 캐릭터·마스코트 부재
- 현 이모지: ⚙ ? 🎨(갤러리) ▶ ↻ — 기능 아이콘 위주, 감정 아이콘 거의 없음.
- 마스코트 캐릭터(연금술사 꼬마, 붓 요정 등) 0개 → 감정 이입 대상 부재.
- 성공/실패 피드백이 숫자와 별점뿐 — 아이들은 "칭찬하는 표정"을 기대.

## B. 재설계 원칙 — 아이 친화 3축

### B-1. 큰 터치 타깃 (Big Touch Target)
- **최소 60pt** 모든 인터랙티브 요소 (기존 44pt에서 상향).
- **CTA 80pt 이상** — "재시작", "다음 스테이지", "로비로" 등 주요 버튼.
- **간격 8pt 이상** — 오탭 방지 (Material 8dp, 우리는 12pt 권장).
- **드롭존 확장** — 팔레트 슬롯 히트박스 CellSize × 1.15 (시각 크기와 별개).

### B-2. 명확한 어포던스 (Clear Affordance)
- **모든 슬롯·버튼에 라벨** — "이게 뭔지" 3초 이내 이해되도록 아이콘 + 텍스트 병기.
- **빈 상태 = "+" + pulse** — 슬롯이 비어있음을 "채울 수 있다"로 번역.
- **채워진 상태 = 블록 + "→ 꺼내기" 힌트** — 드래그 가능함을 미세한 화살표 표시로.
- **드래그 중 drop target 강조** — 대상 셀을 밝기 +15% + 가벼운 링.
- **비활성 버튼은 확실히 죽인다** — opacity 0.35 + 자물쇠 아이콘.

### B-3. 활기찬 톤 (Playful Tone)
- **pastel 배경 + 채도 높은 블록** — 배경은 부드럽게, 주인공(블록)은 또렷하게.
- **둥근 모서리 전면 적용** — radius 12~24px, 직각 금지.
- **이모지 3종 원칙** — 기능/목표/감정 각 1종, 과용 금지.
- **친화적 문구** — "실패"→"다시!", "경고"→"조심!", "초기화"→"처음부터 시작할까?"
- **마스코트 등장** — 결과 화면/로비 중앙에 작은 연금술사 캐릭터 (Phase 1 이후 아트 필요, 우선은 이모지 🧪🧙‍♀️로 대체).

| **감정** | 🎉 💧 👆 | 축하 / 타이틀 / 튜토리얼 가이드 |

## G. 애니메이션 톤

| 이벤트 | 현재 | 재설계 |
|---|---|---|
| 버튼 hover/press | 없거나 GUI 기본 | **scale 1.00 → 1.05 (80ms ease_out_quart) + brightness +8%** |
| 팔레트 빈 슬롯 "+" | 없음 | **scale ±5%, 2s sine loop** (숨쉬듯이) |
| 결과 별 등장 | scale pop | **scale pop + 반짝이 파티클 8개 (각 0.6s 수명)** |
| 블록 픽업 | scale up | **유지** + jelly squash(0.95, 1.05, 1.00) |
| 혼합 성공 | 섬광 | **유지** + 중앙 화면 heart pulse (1회, 200ms) |
| 프롬프트 달성 | 없음 | **프롬프트 바 전체 green pulse 400ms** + 🎉 파티클 |
| 실패/무효 drop | 스냅백 | **스냅백 + shake 4px 3회** (100ms) |

**원칙**: 기존 Motion 6종 easing(§6.1) 를 그대로 사용. Reduce Motion 설정 시 duration × 0.3 + overshoot 제거(기존 규약 유지).

## H. 즉시 착수 TOP 5 (다음 라운드 투입)

리더가 다음 라운드 Coder Teammate 에게 전달할 **최고 ROI** 5개:

### TOP-1. 팔레트 슬롯 재설계 (검은 상자 문제 해결)
- `ResetPaletteColors()` 빈 슬롯 색 `#D4BFFF @ 0.18` 로 교체.

- 채워진 슬롯 하단 `↑ 꺼내기` 캡션 12pt.
- 슬롯 크기 CellSize × 0.85 → × 0.95.
- **예상 ROI**: 최고. 유저 최대 불만 해결. 작업 반나절 이내.

### TOP-2. 모든 주요 버튼 60pt+ / CTA 80pt+ 확대
- 로비 ⚙/? → 60×60
- 일시정지/재시작/다음 → 80h pill (radius 24)

### I-2. 빈 슬롯 "+" 표기
```csharp
for (int i = 0; i < PaletteCount; i++) {
  if (_paletteColors[i] != ColorId.None) continue;
  var sp = Camera.main.WorldToScreenPoint(_palettePos[i]);
```

## J. 리스크 / 제약

1. **OnGUI 한계**: radius, 그라디언트, 점선은 GUI 기본으로 구현 불가 → 근사 + 별도 Texture2D 생성. 완전한 해결은 UGUI/UI Toolkit 마이그레이션 이후 (Phase 1).
2. **이모지 플랫폼 차이**: iOS SF Emoji vs Android Noto Emoji 렌더링 차이. 핵심 이모지는 **TMP SDF Atlas에 사전 베이크**하여 일관성 확보 권장.
3. **폰트 라이선스**: Pretendard/Inter 는 SIL OFL 1.1 → 상업 가능, 변경 없음.
4. **접근성 트레이드오프**: pastel 배경은 보기 좋지만 WCAG 대비비 재검증 필요. 특히 보라 `#9D4EDD` on `#2E2A5C` 는 명암비 3.0 미만 우려 — 흰 stroke 1px 추가 권장.
5. **마스코트 아트 미확보**: Phase 1 D6 까지는 이모지 🧪🧙‍♀️로 대체. 일러스트 외주 타임라인은 별도 Doc Writer 협의.
