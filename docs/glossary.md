# 팀 공용 용어집 (Glossary)

> 컬러 믹스: 연금술사 프로젝트에서 사용하는 모든 용어의 단일 출처(Single Source of Truth).
> 신규 용어 추가 시 반드시 이 문서에 등록하고, 각 Teammate는 본 용어를 기준으로 커뮤니케이션합니다.

---

## 1. 색 시스템 용어 (Color System)

### 1차색 (Primary Color)
혼합으로 만들 수 없는 가장 기본이 되는 색. 본 게임에서는 RYB 모델을 따르며 **빨강(Red) / 노랑(Yellow) / 파랑(Blue)** 3종으로 정의합니다.

### 2차색 (Secondary Color)
두 개의 1차색을 1:1로 조합하여 생성되는 색. **주황(Orange, R+Y) / 초록(Green, Y+B) / 보라(Purple, R+B)** 3종.

### 3차색 (Tertiary Color)
1차색과 인접한 2차색을 조합해 만드는 6종의 중간색 (예: 연두(Y+G), 청록(B+G), 자홍(R+P) 등). 후반 스테이지 및 특수 기믹에서 등장합니다.

### 프리즘 블록 (Prism Block)
매치 시 주변 블록을 무지개 색으로 분산 변환시키는 특수 블록. 고연쇄(Chain)의 트리거로 활용됩니다.

### 필터 벽 (Filter Wall)
특정 색만 통과시키는 반투명 장애물. 올바른 색을 매치해야 벽이 사라지며, 색 조합 전략의 핵심 제약으로 작용합니다.

### 회색 블록 / 색 도둑 (Gray Block / Color Thief)
주변 블록의 색을 흡수해 무채색으로 만드는 위협 요소. 방치하면 보드가 회색화되어 게임 오버로 이어질 수 있습니다.

### 감염 (Infection)
회색 블록 또는 특수 방해 블록이 턴마다 주변 칸으로 퍼지는 현상. 플레이어는 감염 확산 전에 차단 매치를 만들어야 합니다.

## 2. 게임 메커닉 용어 (Game Mechanics)

### 연쇄 (Chain / Cascade)
한 번의 매치가 새로운 매치를 연속으로 유발하는 것. 연쇄 횟수에 비례해 점수 배율과 특수 블록 드롭 확률이 증가합니다.

### 팔레트 슬롯 (Palette Slot)
플레이어가 수집한 색을 보관하는 인벤토리형 슬롯. 스테이지 내에서 저장된 색을 보드에 소환하여 돌파 수단으로 사용합니다.

### 프롬프트 (Prompt)
스테이지 시작 시 제시되는 클리어 조건 또는 추가 도전 과제 (예: "30턴 안에 보라색 10개 생성"). 성공 시 별 보상 또는 배지가 지급됩니다.

### 일일 퍼즐 (Daily Puzzle)
매일 갱신되는 고정 시드의 단일 스테이지. 전 플레이어가 동일 보드를 플레이하며 리더보드로 점수 경쟁.

### 명화 복원 (Masterpiece Restoration)
스테이지 클리어 시 월드맵의 명화·풍경 퍼즐 조각이 색상과 함께 채워지는 메타 진행 시스템. 게임의 내러티브 동력.

---

## 3. 개발 프로세스 용어 (Dev Process)

### Architect
프로젝트의 리더 Agent. 시스템 설계, 기술 의사결정, 교차레이어 조정의 최종 책임을 가집니다.

### Teammate
Architect 외 전문 역할 Agent의 총칭 (UX / Coder / Debugger / Test / Performance / Doc). 각자 병렬로 작업하며 Architect와 동기화합니다.

### 과학적 토론 (Scientific Debate)
의사결정 시 각 Teammate가 **가설 → 근거 → 반박 → 결론**의 구조로 논의하는 프로젝트 규약. `process.md`에 모든 토론 로그가 기록됩니다.

### 교차레이어 조정 (Cross-Layer Coordination)
한 Teammate의 결정이 다른 레이어(예: UX 변경이 Performance에 영향)에 파급될 때 Architect 주도로 영향 평가 및 합의를 거치는 프로세스.

## 4. 기술 용어 (Technical)

### Metaball
서로 가까운 구체가 유기적으로 합쳐져 보이게 하는 쉐이더 기법. 색 조합 / 액체성 폭발 이펙트 연출에 사용됩니다.

### Jelly Sprite
스프라이트에 물리 기반 탄성 변형을 주는 기법. 블록이 탭·매치·폭발 시 젤리처럼 출렁이게 하여 촉감적 피드백을 강화합니다.

### 오브젝트 풀링 (Object Pooling)
블록·이펙트 등 빈번히 생성/파괴되는 오브젝트를 미리 생성해두고 재활용하는 최적화 기법. GC 부하 감소와 프레임 안정화가 목적입니다.

### SRP Batcher (Unity)
Unity Scriptable Render Pipeline의 드로우콜 최적화 배처. 동일 쉐이더 변종의 렌더링을 일괄 처리하여 2D 퍼즐의 다수 스프라이트 렌더 비용을 크게 절감합니다.

### Riverpod Provider (Flutter 후보)
Flutter 생태계의 상태관리 라이브러리. Flame 엔진 채택 시 게임 상태·세이브·UI 간 반응형 바인딩 레이어로 사용됩니다.
