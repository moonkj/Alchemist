## 0. 개요 및 범위

본 문서는 "컬러 믹스: 연금술사" v1.0.0 출시 직후부터 v1.2 공개 베타까지의 **QA 전략, 플레이테스트 프로토콜, KPI, 자동화 시나리오**를 단일 기준으로 정리한다. 개발 현황 (EditMode 120+ 케이스, Phase 4 Juice & Polish 완료) 을 전제로 하며, 도메인 결정론 원칙을 유지한 상태에서 통합/시나리오/성능/접근성 테스트를 확장한다.

- **기준 엔진**: Unity 2022 LTS (iOS 우선, Android 병행)
- **기준 디바이스**: iPhone SE 2 (저사양), iPhone 13 (중급), iPhone 15 Pro (고사양), Galaxy A32 (Android 저사양)
- **기준 FPS**: 60fps 유지 (저사양 45fps 허용)

## 1. 테스트 피라미드 (현황 + 목표)

### 1.1 현황 (v1.0.0)

| 계층 | 현황 | 커버리지 추정 |
|---|---|---|
| EditMode (단위/도메인) | **120+ 케이스** | Domain Core 약 85% |
| PlayMode (통합) | 0본 | 0% |
| End-to-End (시나리오) | 수동 체크리스트만 | N/A |
| 성능/프로파일 | 1회성 수동 | N/A |

### 1.2 v1.1 목표 (출시 4~6주 내)

- **PlayMode 통합 스모크 5본 추가**
  1. 부트 → 로비 → 스테이지 1 진입 → 첫 턴 실행 성공
  2. 스테이지 클리어 → 갤러리 → 배지 반영 확인
  3. 스테이지 실패 → 리트라이 → 상태 초기화 확인
  4. 앱 백그라운드 ↔ 포그라운드 복귀 시 세이브 무결성
  5. 튜토리얼 전체 통과 (스킵/정상 경로 각 1회)

- **결정론 Golden Replay 3본**
  - 입력 시퀀스 → 보드 해시 스냅샷 비교
  - 시드 고정 스테이지 3종 (쉬움/보통/어려움)
  - 회귀 시 임의 규칙 변경 감지

### 1.3 v1.2 목표 (오픈 베타 전)

- **시나리오 자동화 20본** (본 문서 §8 참조)
- PlayMode 통합 스모크 10본으로 확장
- Golden Replay 10본 확장 (특수 블록 전부 포함)
- 성능 자동 회귀 (frame time, GC alloc, draw call 기준선 벤치)

### 1.4 DoD (테스트 피라미드)

- EditMode 실패 0건 (CI 차단)
- PlayMode 스모크 5본 pass rate 100%
- Golden Replay hash 불일치 시 빌드 불가 플래그

## 2. Playtest 프로토콜

### 2.1 내부 테스트 (개발자 셀프)

**체크리스트 (매 빌드당 실행)**

| # | 시나리오 | 반복 | 확인 항목 |
|---|---|---|---|
| A1 | 스테이지 클리어 | 3회 | 점수/배지/갤러리/사운드/햅틱 |
| A2 | 스테이지 실패 (이동 초과) | 3회 | 리트라이 플로우, 상태 초기화 |
| A3 | 스테이지 이탈 (홈 복귀) | 3회 | 진행 저장 유무, 재진입 상태 |
| A4 | 튜토리얼 | 2회 | 정상/스킵 경로 모두 |
| A5 | IAP 시뮬레이터 | 2회 | 잉크 구매, 복원, 에러 핸들링 |

- **DoD**: 15개 셀 전부 통과 + 크래시 0건
- **Exit Criteria**: P0/P1 버그 0건 → 클로즈드 베타 진입

### 2.2 클로즈드 베타 (TestFlight 50명)

- **모집 기준**: 퍼즐 경험자 60% + 캐주얼 40%, 성별 50:50, iOS 15+ 보유
- **기간**: 2주, 주 1회 설문 (NPS, 난이도 주관 점수)
- **수집**: Firebase Analytics + Crashlytics + 주관 피드백 폼
- **DoD**: 평균 세션 ≥ 5분, 크래시 프리 유저 ≥ 99%, NPS ≥ 30
- **Exit Criteria**: D1 리텐션 ≥ 30%, P0 0건, P1 ≤ 3건 (수정 완료)

### 2.3 오픈 베타 (500명 규모)

- **플랫폼**: TestFlight (iOS) + Google Play Internal Testing (Android)
- **기간**: 3~4주
- **Firebase Analytics 이벤트 10종** (§3.3 참조)
- **DoD**: D1 ≥ 35%, D7 ≥ 15%, 크래시 프리 ≥ 99.5%
- **Exit Criteria**: 스토어 리뷰 지표 준수 + 메타 스토어 자동 심사 통과 → 정식 출시

### 3.2 리텐션 / 참여도

| 지표 | 목표 v1.1 | 목표 정식 |

| 지표 | 목표 |
|---|---|

## 5. 성능 QA 체크리스트

### 5.1 저사양 기기 연속 플레이

**대상**: iPhone SE 2 (A13), Galaxy A32 (Helio G80)
**시나리오**: 스테이지 1 → 20 연속 플레이, **15분 지속**

| 항목 | 허용 기준 |
|---|---|
| 평균 FPS | ≥ 45fps (SE 2) / ≥ 40fps (A32) |
| 1% low FPS | ≥ 30fps |
| 프레임 드롭 구간 | 전체의 ≤ 5% |
| CPU 표면 온도 | ≤ 42°C (SE 2 기준) |

### 5.2 메모리 누수

- **도구**: Unity Memory Profiler
- **방법**: 스테이지 진입/이탈 10회 반복, 5분 간격 Snapshot 3회 캡처
- **허용 기준**: 스테이지 이탈 후 메모리 원복 델타 ≤ 5MB, 누적 증가 ≤ 15MB

### 5.3 배터리 소모

- **측정**: 15분 연속 플레이 당 % 감소량
- **허용 기준**: iPhone SE 2 ≤ **8%** / Galaxy A32 ≤ 10%
- **초과 시**: GPU 이슈 의심 → VFX 드로우콜/오버드로우 점검

### 5.4 빌드 크기

- iOS IPA ≤ **150MB** (초기), 에셋번들 분리 시 ≤ 80MB
- Android AAB ≤ 120MB

## 6. 접근성 QA

### 6.1 VoiceOver / TalkBack 호환성

- 메인 UI 라벨 100% 커버
- 보드 내 블록 상태 음성 안내 (선택 블록 색/연결 개수)
- 이동 완료 후 잔여 이동 수 자동 안내

### 6.2 색맹 모드

- **대상**: Deuteranopia, Protanopia, Tritanopia 3종
- **검증**: Sim Daltonism (macOS) 로 시뮬레이션 캡처 → 5색 구분 가능 여부
- **보조 수단**: 블록 상단 심볼 오버레이 (△□○◇★) 활성화 체크
- **스크린 리딩 테스트**: 색맹 모드 ON 시 VoiceOver 라벨에 "빨강/파랑" 대신 "심볼명" 포함

### 6.3 한손 조작 도달성

- **기준**: iPhone 15 Pro Max (6.7") 오른손 엄지 기준
- **도달 맵**: 주요 버튼 (팔레트/아이템/메뉴) 모두 하단 70% 영역 배치
- **체크**: 스테이지 진입 후 필수 액션 3종 모두 한손 내 도달 가능

## 7. 로컬라이제이션 QA

### 7.1 언어 우선순위

1. **한국어** (런치 기본)
2. **영어** (런치 동시)
3. 일본어 (v1.1)
4. 중국어 번체 (v1.2)
5. 터키어/독일어/스페인어 (v1.3, 길이 변동 검증 우선)

### 7.2 텍스트 길이 변동 체크리스트

| 언어 | 한국어 대비 길이 변동 | 주요 위험 |
|---|---|---|
| 영어 | 약 +20% | 버튼 라벨 2줄 넘침 |
| 독일어 | 약 +40% | 복합명사로 인한 overflow |
| 터키어 | 약 +35% | 교착어 특성, 접미사 |
| 일본어 | 약 -10% | 폰트 렌더링 검증 필요 |
| 중국어 간체 | 약 -30% | 행간/자간 조정 |

- 모든 UI는 **+40% 길이 허용**을 가정한 유연 레이아웃
- 고정 폭 버튼 사용 금지, FlexContainer 필수

- 보드 해시 스냅샷: `BoardState.ToDeterministicHash()` 활용

**DoD**
- 20본 전부 CI 환경에서 자동 실행 가능
- 1회 수행 시간 ≤ 15분
- 실패 시 Slack / GitHub Actions 알림 자동 트리거
