### 3.1 상태 변화
- **Pause Overlay**: 반투명 backdrop `overlay.dark` + 중앙 카드 (재개/재시작/나가기 3 버튼).
- **Hint Overlay**: 롱프레스 1s → 후보 셀 0.3α pulse (motion §1 `PreviewTarget`).
- **Fail Soft Modal**: 이동 0 도달 시 "이동 +5 구매 또는 종료" — 토스트가 아닌 모달 (elevation_4).
- **로딩**: 보드 영역만 붓-스트로크 Lottie 중앙 정지 (1.2s).
- **에러**: 게임 중 네트워크 장애 — HUD 우상단 ☁️ 슬래시 아이콘 (클릭으로 설명 toast).
- **빈**: 없음 (항상 보드 존재).

### 3.2 컴포넌트 재사용
HUD(IconButton × 3 + Label × 2), PromptBanner × 1, Board × 1, PaletteSlot × 3, ItemButton × 3, Toast × 1.

### 3.3 다크 모드
- 보드 배경 `neutral.800`, 셀 기본 `neutral.700`.
- 블록 색상은 **light variant** 사용 (#F27685 등) → 다크 배경 위 대비 확보.
- HUD 아이콘 `neutral.200`, 프롬프트 배너 배경 `#1F242D` + yellow.base border.

### 3.4 Safe Area
- 상단 HUD 자체에 `top-safe + 8px` 마진 → Dynamic Island 회피 (아이폰 14 Pro).
- 하단 팔레트/아이템 row 는 `bottom-safe + 16px` 아래 여유.

### 3.5 한손 도달성 (390×844)
```
Hot   (엄지 편안, ≤ 화면 하 50%):
  - 팔레트 슬롯 3
  - 아이템 버튼 3
  - 프리뷰 토스트
  - 보드 하단 2행

Warm  (살짝 당김, 50~75%):
  - 보드 중앙 3행
  - Prompt Banner

Cold  (재파지 필요, 75%+):
  - HUD 상단 ⓘ / ⏸ / ☰
  → 대응: 양방향 Swipe-down gesture 로 상단 메뉴 풀다운 제공.
```

## 11. 한손 조작 도달성 맵 (요약)

390×844 기준. 엄지 Hot/Warm/Cold 구역을 모든 화면에 적용.

```
y = 0        ┌──────────┐
             │   COLD   │  0~30% (0~253pt) — 뒤로가기, 타이틀만
y = 253      ├──────────┤
             │   WARM   │  30~55% (253~464pt) — 보조 액션, 배너, 탐색
y = 464      ├──────────┤
             │   HOT    │  55~90% (464~760pt) — 주요 CTA, 팔레트, 탭바
y = 760      ├──────────┤
             │ SAFE BOT │  90~100% (760~844pt) — iOS 홈 인디케이터
y = 844      └──────────┘
```

**원칙**
1. 주요 CTA (모험 시작 / 시작 / 다음) 는 HOT 내.
2. 파괴적 액션 (계정 삭제, 결제 확정) 은 Cold+대형 모달로 "의도적 재파지" 유도.
3. 팔레트/아이템 row 는 무조건 HOT 하단 — 세션 중 최빈 조작.
4. 좌상단 뒤로가기는 Cold — 대안으로 **swipe-right edge gesture** 전역 제공.
