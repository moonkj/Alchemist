## 1. 핵심 유저 여정 (User Journey)

대상: iOS/Android 매치-3 카테고리에 익숙한 10~40대 모바일 유저. 목표는 **첫 세션에서 "색 혼합의 쾌감"을 체득시키는 것**.

### 1.1 첫 0~30초 (Hook)
| 시점 | 유저 상태 | 화면 / 이벤트 | UX 목표 |
|---|---|---|---|
| 0s | 앱 탭 | 스플래시 (흑백→컬러 번짐 애니 1.5s) | "색이 돌아온다" 감정 선언 |
| 1.5s | 기대 | 무채색 로비 + 중앙 '붓' 아이콘 1개만 강조 | 결정 피로 0 |
| 3s | 첫 탭 | 튜토리얼 Stage 0 자동 진입 (스킵 불가) | 학습 강제 확보 |
| 5s | 관찰 | 3x3 미니보드 + 빨강·파랑 2블록만 존재, "빨+파를 만나게 하세요" 문구 | 목표 최소화 |
| 10s | 첫 조작 | 손가락 가이드 애니(점선) → 빨강을 파랑 위로 이동 | 단일 조작만 학습 |
| 15s | 첫 성공 | 보라색 생성 + 화면 섬광 + 햅틱(Medium) + "보라!" 라벨 | 도파민 트리거 |
| 22s | 2번째 성공 | 팔레트 슬롯 해제, 빨강 한 개를 슬롯에 저장 권유 | 메커니즘 확장 |
| 30s | 각인 | "당신은 연금술사입니다" 타이틀 등장 + 로비로 복귀 | 정체성 부여 |

**성공 지표(UX)**: 30초 구간 이탈률 < 8%, 첫 혼합 성공률 > 95%.

### 1.2 첫 30초~3분 (Activation)
1. 로비에 **"챕터 1: 잃어버린 노을"** 1개만 언락. 다른 버튼은 자물쇠.
2. Stage 1 진입 → 5x6 보드, 목표 프롬프트: "보라 3개 만들기, 이동 10회".
3. 첫 연쇄(보라+노랑=검정) 발생 시 **슬로우모션 0.4s + 전체 보드 흔들림**.
4. 클리어 → 결과 화면 **"노을 조각 1/12 복원"** 갤러리 티저.
5. Stage 2 진입 전 **잉크 에너지 개념** 1회 설명 (모달, 탭 2회로 닫힘).

### 1.3 첫 세션 완주 (5~8분)
- Stage 1~3 (튜토리얼 유닛) → 갤러리 1장의 1/4 복원 → **보상 모달(배지 1개 + 프리즘 아이템 1)**.
- 이탈 트리거 방지: 세션 종료 전 "내일 새 프롬프트 도착" 푸시 옵트인을 **Stage 3 완료 직후 1회만** 노출.
- 재진입 Hook: 일일 퍼즐 카드 + 랭킹 상위 10% 프리뷰.

## 4. 조작 UX 결정안

### 4.1 A안 드래그 앤 드롭 vs B안 스와이프 — 과학적 토론

**전제 — 색 혼합 게임의 핵심 요구사항**
1. 유저가 "어느 블록을 어느 블록과 섞을지" **의도적으로 지정**해야 한다 (매치-3의 우연성과 다름).
2. 팔레트 슬롯에 **저장/꺼내기** 조작이 필요하다.
3. 3차 연쇄가 터지기 전 **미리보기**를 보여줘야 한다 (§7 진입장벽).

| 기준 | A. Drag & Drop | B. Swipe (인접 스왑) |
|---|---|---|
| 의도성 | ★★★ 원하는 타겟 지정 가능 | ★☆☆ 인접 셀만 가능 |
| 팔레트 슬롯 연동 | ★★★ 보드→슬롯 드래그 자연스러움 | ★☆☆ 별도 탭 UI 필요 |
| 미리보기 타이밍 | ★★★ 드래그 중 hover 상태에서 표시 | ★★☆ 스왑 직전 짧게 표시 |
| 손가락 피로도(세션 10분) | ★★☆ 이동거리 김 | ★★★ 짧은 제스처 |
| 한손 조작(대형 디바이스) | ★☆☆ 엄지 도달 거리 문제 | ★★★ 우수 |
| 매치-3 관습 준수 | ★☆☆ 신규 패턴 학습 필요 | ★★★ Candy Crush 관습 |
| 오조작 (fat finger) | ★★☆ 드롭 타겟 명확화 필요 | ★☆☆ 원하지 않는 방향 스와이프 위험 |
| 색 조합 표현력 | ★★★ 3개 이상 연속 합성 가능 | ★★☆ 1회 스왑 단위로 제약 |
| 학습 비용 | 중간 (5초 튜토리얼) | 낮음 (3초) |
| 60fps 부하 | 드래그 중 지속 리렌더 | 이벤트 기반, 가벼움 |

**토론**
- **B안의 강점**은 "기존 유저의 무학습 진입"이다. 그러나 본 게임의 **IP 차별점은 "원하는 색을 의도적으로 섞는 연금술"** 이므로, 인접 셀만 섞을 수 있는 B안은 **핵심 판타지를 희석**한다.
- **A안의 약점**인 손가락 피로도/도달 거리는 **보드 크기를 6×7 이하로 제한**하고, **팔레트 슬롯을 하단 중앙**에 배치해 엄지 이동 범위를 최소화하면 완화된다.
- 미리보기(§7)는 **드래그 중에만 비차단적**으로 띄울 수 있다. B안의 짧은 제스처 구간에서는 미리보기를 자연스럽게 표시할 타이밍이 부족하다.
- Fat finger 이슈는 A안에서 **드롭 타겟 주변 16px 확장 히트박스 + 스냅 애니**로 해결 가능. B안은 방향 오인식이 근본적으로 해결되지 않음.

**권고안: A안 Drag & Drop 채택**
- 단, **짧은 드래그(= 인접 1칸)** 는 스와이프처럼 동작하도록 **하이브리드**로 구현한다.
- 드래그 시작 후 16px 이내에서 손가락을 떼면 → 인접 방향 스왑으로 해석.
- 드래그 거리 > 16px 이면 → 일반 드래그 앤 드롭 (임의 셀/슬롯으로 이동).

### 4.2 햅틱 / 사운드 피드백 타이밍

| 이벤트 | 햅틱 | 사운드 | 비고 |
|---|---|---|---|
| 블록 픽업 (드래그 시작) | Selection (Light) | 소리 없음 | 과다 방지 |
| 유효한 드롭 타겟 hover | Soft tick | 저음 '톡' | 첫 방문 3회만 |
| 1차 혼합 성공 (2차색 생성) | Medium Impact | 벨 (G4) | 핵심 피드백 |
| 2차 연쇄 (2차→3차 검정/흰) | Heavy Impact + 진동 0.3s | 코드 (CMaj7) | 특별감 |
| 프롬프트 달성 순간 | Success (iOS Notification) | 팡파레 1.2s | 결과 화면 전환 전 |
| 잘못된 드롭 (무효) | Warning (Rigid) | 바람빠지는 '쉿' | 길이 0.1s |
| 턴 부족 경고 (≤3턴) | 없음 | 심장박동 앰비언트 | 루프 |
| 잉크 부족 | Warning | 없음 | 모달로 강화 |

**원칙**
- 세션 당 햅틱 트리거 수 ≤ 80회 (과다 시 배터리/피로).
- 설정에서 3단계(풍부/기본/끔) 제공.
- 사운드는 **BGM과 SFX 버스를 분리**해서 BGM 음소거 시에도 성공 사운드는 남도록.

## 5. 상태 UX (Empty / Loading / Error)

### 5.1 빈 상태
- **갤러리 비어있음 (첫 진입)**: 흑백 액자 3개 + "첫 스테이지를 깨면 이곳이 채워집니다" + [모험 시작] 버튼. 액자가 미세하게 흔들리는 Lottie.
- **랭킹 비어있음 (일일 진입 직후)**: "오늘의 순위는 첫 도전 후 집계됩니다" + [일일 퍼즐로] 버튼.
- **배지 비어있음**: 회색 실루엣 24개. 탭 시 "조건: ???" (힌트 3글자만).

### 5.2 로딩
- **스테이지 진입 로딩 (≤ 1.5s)**: 붓으로 화면을 쓸어 원색이 번지는 Lottie (중앙 고정). 텍스트 없음.
- **로딩 > 3s**: "색을 섞는 중..." 텍스트 추가. 5s 초과 시 팁 카드 노출.

### 5.3 에러
- **네트워크 에러**: 토스트 하단, "오프라인 모드로 계속합니다" + [재시도] 링크. 로비/스테이지는 동작, 랭킹/상점/일일만 잠금.
- **결제 에러**: 모달, "다시 시도" + "고객센터" 2개 버튼. 에러 코드 작게 노출(복사 가능).
- **세이브 충돌 (두 기기)**: Blocking 모달, 최근 기기 세이브 선택 강제.

### 5.4 잉크 부족
- 스테이지 진입 시점 체크 → 모달 "잉크가 부족해요" + 3옵션: [5분 후 1잉크 회복 대기] / [광고 시청 +1] / [상점].
- **게임 중 소모 없음** (입장 시 1회 소모). 중간 광고/모달은 금지. UX 원칙: **플레이 리듬 방해 금지**.

### 5.5 오프라인
- 상단바에 ☁️ 슬래시 아이콘. 탭하면 "오프라인 — 기록은 온라인 복귀 시 동기화" 툴팁.
- 챕터 모험 전체 플레이 가능, 일일/랭킹/상점은 비활성(회색 + 자물쇠).
- 복귀 시 백그라운드 동기화 → 성공하면 오른쪽 상단 🟢 체크 0.8s 후 페이드아웃.

## 7. 진입 장벽 저감 장치

### 7.1 "이거 섞으면 뭐 되지?" 미리보기
- **실시간 프리뷰 토스트** (§3.1): 드래그 중인 블록이 접촉 후보 셀 위에 올라가면, 하단 32px 토스트에 `"🔴 + 🔵 = 🟣 보라"` 형태로 표시.
- **색상환 미니맵**: 설정에서 켤 수 있는 보조 위젯. 하단 우측에 48×48 원형 색상환, 탭 시 확대되어 혼합 규칙 매트릭스 노출.
- **롱프레스 힌트**: 블록을 1초 이상 누르면 그 블록이 섞일 수 있는 인접 셀 후보를 0.3 alpha 펄스로 하이라이트.

### 7.2 튜토리얼 설계 초안
- **3단계 스캐폴딩 원칙** 적용.
  - **Stage 0 (강제, 30초)**: 1개 조합만 (빨+파=보라). 다른 블록 없음. 실패 불가능.
  - **Stage 1 (권장, 90초)**: 첫 슬롯 등장. 시스템이 슬롯 사용 타이밍을 "권유"(점선 화살표), 강제는 아님.
  - **Stage 3 (첫 연쇄, 120초)**: 보드 가득. 2차 연쇄(보라+노랑=검정)가 **자연스럽게** 터지도록 레벨 디자인. 터진 직후 슬로우 모션 + "연쇄!" 라벨.
- **진행형 공개(Progressive Disclosure)**: 회색/프리즘/필터벽은 챕터 2부터. 한 번에 한 개념만.
- **실패에 관대**: 첫 챕터 전 스테이지는 **이동 +5 소프트 페일** 1회 무료 제공.
- **스킵 가능 지점**: Stage 0만 강제, 이후 "이미 해봤어요" 버튼 → 챕터 2 모의 시험으로 직행.

## 9. UX가 확정 전까지 유보한 항목

1. **색맹 모드 세부 규격** — 모양/패턴 오버레이를 블록마다 입힐지, 색상환만 교체할지 보류. 접근성 리서치 후 결정.
2. **랭킹 UI의 "친구" 정의** — OS 연락처 / 게임 센터 / 자체 친구 시스템 중 어떤 것을 연동할지 플랫폼/백엔드 결정 대기.
3. **갤러리 공유 이미지 포맷** — 스토리형(9:16) vs 스퀘어(1:1) 이중 지원 여부. 마케팅 요구 수신 후.
4. **PVP / 비동기 대결 모드 존재 여부** — 로드맵 상 Phase 2 이후로 보이나, 로비 하단바에 자리 비울지 결정 보류.
5. **세션 끝 리워드 광고(Rewarded Ad)** 노출 빈도/위치 — BM 팀 정책 대기.
6. **가로 모드(landscape) 지원 여부** — iPad/폴더블 대응. 1차는 세로만, 추후 결정.
7. **오프라인 진행 상한** — 며칠 치 진행을 오프라인으로 허용할지, 안티치트 정책과 동기화 필요.
8. **팔레트 슬롯 최대 개수** — 3개로 상정했으나, 게임 디자인 밸런스 결과에 따라 2~4 재조정 여지.
9. **프리뷰 토스트 on/off 기본값** — 초보 3스테이지까지 ON 이후 OFF 권고. A/B 테스트 후 확정.
10. **햅틱 강도 기기별 커브** — Android 벤더별 진동 모터 특성 리서치 필요.
